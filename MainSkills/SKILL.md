# SkillManager

All skills are stored under `I:\.agents\MainSkills\skills\`.
Read `skills/skill_index.yaml` first, then load only the skills relevant to the task.

## Startup Procedure

```
1. Read this file (I:\.agents\MainSkills\SKILL.md)
2. Read I:\.agents\MainSkills\skills\skill_index.yaml
3. Identify relevant skills by tags and description for the current task
4. Load only relevant skills from I:\.agents\MainSkills\skills\[skill_name]\SKILL.md
5. Do not read unrelated skills
6. Reference I:\.agents\GlobalRules\GLOBAL_RULES_EN.md only when skills cannot resolve the issue
```

## Skill Search Method

1. Read `skills/skill_index.yaml` for the full skill list
2. Match task keywords against `tags` and `description` fields
3. When multiple skills match, check `priority` (lower = higher priority)
4. Load matched skills from `skills/[skill_name]/SKILL.md`

## Index Maintenance

- `skills/skill_index.yaml` is auto-generated by `skills/generate_skills.bat`
- Manual editing is prohibited
- Always run `generate_skills.bat` after adding or modifying a skill

## Folder Structure

```
I:\.agents\
  GlobalRules\
    GLOBAL_RULES_EN.md          ← English original (last resort reference)
    rules\
      CORE.md                   ← Always read at startup
      detail\
      examples\
      lang\
      scale\
  MainSkills\
    SKILL.md                    ← This file (always read first)
    skills\
      skill_index.yaml          ← Skill list (auto-generated, do not edit)
      generate_skills.bat       ← Regenerate skill_index.yaml
      generate_skills.ps1
      [skill_name]\
        SKILL.md
```

## SKILL.md Format

```yaml
---
name: SkillName
description: When to use this skill (used by agent to judge relevance)
priority: 1
tags: [tag1, tag2]
---

# Skill content here
```
